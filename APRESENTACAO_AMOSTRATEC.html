1. O que é o projeto

O MedSystem PRO é um sistema web completo para gestão de consultório/clínica. Centraliza agenda, prontuário eletrônico, exames, prescrições, diagnósticos (CID-10), financeiro, relatórios, equipe e mensagens internas — tudo integrado em uma única aplicação.

"Desenvolvemos um sistema clínico funcional e o publicamos na AWS com Terraform: um único terraform apply sobe rede, banco privado, API, frontend no S3 e HTTPS via CloudFront."

VPC — Rede

ComponenteConfiguração
VPC10.0.0.0/16 — DNS habilitado
Subnet pública10.0.1.0/24 (us-east-1a) → EC2
Subnets privadas10.0.2.0/24 e 10.0.3.0/24 → RDS (2 AZs)
Internet GatewayInternet só pela subnet pública

Security Groups (Firewall)

EC2 (Backend): porta 5000 liberada apenas para IPs do CloudFront (prefix list AWS). SSH na porta 22 para gerenciamento.
RDS (Banco): porta 3306 liberada somente para o Security Group da EC2 — o banco não é acessível pela internet.

6. Deploy automatizado (Terraform)

Um único terraform apply executa todo o fluxo:

  1. Cria VPC, subnets, security groups e rotas
  2. Sobe RDS MySQL privado
  3. Sobe EC2: instala dependências, clona GitHub, cria .env, configura systemd
  4. Envia medsystem_banco.sql da máquina local → EC2 via SSH (nunca vai ao GitHub)
  5. EC2 importa o schema no RDS pela rede privada
  6. Inicia o Flask somente após o banco estar pronto
  7. Cria bucket S3, monta index.html (substitui Jinja includes) e publica frontend
  8. Cria CloudFront e exibe URL HTTPS final
Resultado: terraform output → URL https://xxxx.cloudfront.net

7. Banco de dados (destaque técnico)

  • Tabelas: usuarios, medicos, paciente, consulta, exames, prescricoes, diagnosticos, receitas, despesas, mensagens, cid10, logs_auditoria
  • Stored Procedures: cadastrar paciente, agendar/cancelar consulta, registrar prontuário, registrar pagamento
  • Triggers: auditoria automática de alterações
  • Views: agenda do dia, histórico do paciente, financeiro mensal, exames pendentes

8. Segurança

  • HTTPS via CloudFront (acesso do usuário sempre criptografado)
  • S3 privado — leitura apenas pelo CloudFront (OAC)
  • API na EC2 acessível somente via CloudFront
  • RDS sem IP público (subnets privadas)
  • Senhas com hash Bcrypt + autenticação JWT
  • Credenciais, .pem e .sql fora do Git (.gitignore)

10. Perguntas frequentes

PerguntaResposta
Por que separar S3 e EC2?Escalabilidade e custo: estáticos no S3+CDN; lógica na EC2
Por que CloudFront na frente?HTTPS, cache do frontend e controle de acesso à API
Como o banco entra no RDS?Importação pela EC2, dentro da VPC, via rede privada
O que é Terraform?Infraestrutura como código — reproduzível e versionável
Autenticação?JWT no login; rotas protegidas exigem Bearer token
